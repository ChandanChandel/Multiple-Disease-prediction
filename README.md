# 🧑‍⚕️ Multiple Disease Prediction System

A Streamlit web application that predicts the likelihood of **Diabetes**, **Heart Disease**, and **Parkinson's Disease** using machine learning models trained on clinical datasets.

## Features

- **Diabetes Prediction** — based on pregnancies, glucose level, blood pressure, skin thickness, insulin, BMI, diabetes pedigree function, and age.
- **Heart Disease Prediction** — based on age, sex, chest pain type, resting blood pressure, cholesterol, fasting blood sugar, ECG results, max heart rate, exercise-induced angina, ST depression, slope, major vessels, and thalassemia.
- **Parkinson's Disease Prediction** — based on 22 biomedical voice measurements (MDVP jitter, shimmer, HNR, RPDE, DFA, spread, PPE, etc.).
- Simple sidebar navigation between the three prediction tools.
- Pre-trained models loaded via `pickle` — no training required to run the app.

## Getting Started

### Prerequisites

- Python 3.8+
- pip

### Installation

1. Clone the repository
   ```bash
   git clone https://github.com/ChandanChandel/Multiple-Disease-prediction.git
   cd Multiple-Disease-prediction
   ```

2. (Optional) Create and activate a virtual environment
   ```bash
   python -m venv venv
   source venv/bin/activate      # On Windows: venv\Scripts\activate
   ```

3. Install the dependencies
   ```bash
   pip install -r requirements.txt
   ```

### Running the App

```bash
streamlit run app.py
```

The app will open automatically in your browser, or you can navigate to `http://localhost:8501`.

## Usage

1. Launch the app and use the sidebar to select a prediction system: **Diabetes**, **Heart Disease**, or **Parkinson's**.
2. Enter the requested medical/clinical values into the input fields.
3. Click the corresponding test result button (e.g., "Diabetes Test Result") to get an instant prediction.

## Model Training

Each disease model was trained separately in the notebooks under `colab_files_to_train_models/`, using the corresponding CSV dataset in `dataset/`. The resulting models are serialized with `pickle` and saved to `saved_models/`, which is what `app.py` loads at runtime.
